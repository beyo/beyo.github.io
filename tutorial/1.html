Getting Started

In Development

This tutorial targets the dev branch of the framework. This notice will be removed once the refactory and testing phase is done.

This tutorial is intended to give an introduction to using Beyo by creating a simple application using the Hierarchical-Model-View-Controller paradigm.

This tutorial will not focus on persistence, templating, or authentication. But will go in depth into the core of Beyo and the best practices to start a small to large scale application.

By the end you will have a working Beyo application and you can then poke around the code to find out more about how it all works and fits together.

Installation

Some assumptions

It is assumed that you already have installed and configured Node.js. Beyo requires and use extensively the most recent ES6 generators and harmony features. Therefore, this tutorial was written and tested using version 0.11.3.

Requirements

Development Tools

In this tutorial, we will make use of the terminal and will not enforce any IDE or text editor. For this reason, those using Win32 operating systems may have to adapt.

Beyo is built on top of Koa, which sits on top of co. If these modules have been tested and working, then everything should be just fine. In short the following command should execute just fine. 

$ node --harmony -e 'function*f(){}'

Getting Beyo Framework

Beyo should to be installed globally : 

$ npm install -g beyo

The installation may be validated by issuing the following command : 

$ beyo --version
beyo@0.2.1
